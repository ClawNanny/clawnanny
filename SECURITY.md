# Security Policy

We take security reports seriously and appreciate responsible disclosure.

## Reporting a vulnerability

**Please do not open a public GitHub issue for security problems.**  Use one of the private channels below instead.

- **Preferred &mdash; GitHub private security advisory:**&nbsp; Click "Report a vulnerability" on this repo's [Security tab](https://github.com/ClawNanny/clawnanny/security/advisories/new).&nbsp;  This creates a private thread visible only to the maintainers and to you.
- **By e-mail:** to [admin@ClawNanny.com](mailto:admin@ClawNanny.com) with a subject line beginning "Security report:".

When you report, please include:

1. A description of the vulnerability and the impact you believe it has
2. Steps to reproduce, including the ClawNanny version (visible in the About dialog) and your Windows version
3. Any proof-of-concept code or screenshots that help us understand the issue
4. Your name and a contact handle if you'd like to be credited in the fix announcement

## What to expect

- We aim to acknowledge new reports within **3 business days**.
- We will keep you informed as we investigate and prepare a fix.
- Once a fix is released, we will publish a brief advisory crediting the reporter (with permission).

## Scope

This policy covers:

- The ClawNanny installer for MacOS and Windows and the desktop application it installs
- Any binaries attached to releases in this repository

It does **not** cover:

- Vulnerabilities in the upstream open-source projects ClawNanny depends on (OpenClaw, Hermes-Agent, etc.).&nbsp;  Please report those to the respective upstream projects.&nbsp;  We are happy to relay reports if you are unsure where to send them.
- Issues in third-party services you connect ClawNanny to (e.g. Anthropic API, OpenAI API, MCP servers you install yourself)

## Supported versions

Only the latest beta release is supported with security fixes at this time.&nbsp;  Once ClawNanny reaches stable, this section will be updated with a longer support window.&nbsp;
